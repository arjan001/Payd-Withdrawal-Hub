Payouts
Merchant Payouts (Kenya)
Pay to M-Pesa Paybill and Till numbers in Kenya. This is useful for B2B payments, utility bills, and any merchant that accepts M-Pesa business payments.

Supported merchant types
Paybill
Pay to a Paybill number with an account reference. Commonly used for utilities, banks, and large businesses.

Example: Paybill 247247

Till Number
Pay to a Till (Buy Goods) number. Commonly used for retail merchants and small businesses.

Example: Till 5432100

Endpoint
POST
/api/v2/payments
Request parameters
Body Parameters (JSON)
Parameter	Type	Required	Description
username	string	Required	Your Payd account username
amount	number	Required	Amount in KES to pay to the merchant
currency	string	Required	Must be "KES"
phone_number	string	Required	Sender's phone number with country code (e.g., +254700000000)
narration	string	Required	Description of the payment
transaction_channel	string	Required	Must be "bank"
channel	string	Required	Must be "bank"
business_account	string	Required	The Paybill or Till number (business short code) of the recipient merchant
business_number	string	Required	The account number under the Paybill number (not applicable for Till payments - Till numbers do not have account numbers)
callback_url	string	Required	URL to receive webhook notification with the final transaction status
wallet_type	string	Optional	"local" (default) or "USD". When set to "USD", the payout is funded from your USD wallet. The amount is still specified in KES and Payd converts from your USD balance automatically
Paybill vs Till For Paybill payments, business_account is the Paybill number and business_number is the account number under that Paybill. For Till payments, business_account is the Till number and business_number is not needed - you can pass any value.
transaction_channel and channel Both transaction_channel and channel must always be set to "bank" for merchant payouts.
Request examples
This example pays KES 12 to a Paybill merchant:

cURL
JavaScript
Python
bash
Copy
curl -X POST https://api.payd.money/api/v2/payments \
  -u 'YOUR_API_USERNAME:YOUR_API_PASSWORD' \
  -H 'Content-Type: application/json' \
  -d '{
    "username": "your_payd_username",
    "amount": 12,
    "currency": "KES",
    "phone_number": "+254700000000",
    "narration": "Payment for goods",
    "transaction_channel": "bank",
    "channel": "bank",
    "business_account": "247247",
    "business_number": "0000000000000",
    "callback_url": "https://your-server.com/callback"
  }'
Responses
Success Response
The payout request was accepted and is being processed. The success: true here only means the request was accepted - wait for the webhook for the final outcome.

200 OK
Copy
{
  "success": true,
  "correlator_id": "9BD090722409eW",
  "message": "Transaction request sent successfully",
  "status": "SUCCESS",
  "channel": "",
  "amount": 0
}
Merchant payouts return a correlator_id instead of transaction_reference. Both serve as unique identifiers - the webhook will include the transaction_reference.
Error Response
The request failed validation. Returns an error status code (e.g. 400).

400 Bad Request
Copy
{
  "status": 400,
  "success": false,
  "message": "invalid phone number"
}
Webhook response
After the payment is processed, Payd sends a POST request to your callback_url with the final transaction status. The result_code and status together are the source of truth for the transaction outcome.

Payment completed
The third_party_trans_id is the M-Pesa transaction receipt number from Safaricom.

Success Callback
Copy
{
  "transaction_reference": "9BD103739849eR",
  "result_code": 0,
  "remarks": "Successfully processes m-pesa transaction",
  "third_party_trans_id": "UB9C46DH3Q",
  "amount": 12,
  "transaction_date": "2026-02-09 10:37:50.286784611 +0000 UTC m=+448005.605396772",
  "forward_url": "https://your-server.com/callback",
  "order_id": "",

  "customer_name": "",

Payment failed / cancelled
Failure Callback
Copy
{
  "transaction_reference": "9BD104311505eR",
  "result_code": 1,
  "remarks": "Transaction failed with message: [Request Cancelled by user.]",
  "transaction_date": "November",
  "forward_url": "https://your-server.com/callback",
  "order_id": "",
  "success": false,
  "status": ""
}
Important The initial API response only confirms the payout request was accepted. Always wait for the webhook callback to determine the final outcome. Check result_code and status: result_code: 0 + status: "success" means the transaction succeeded.
