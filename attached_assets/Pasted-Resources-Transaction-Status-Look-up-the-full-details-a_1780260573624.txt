Resources
Transaction Status
Look up the full details and current status of any Payd transaction using its transaction reference.

Endpoint
GET
/api/v1/status/{transaction_reference}
Replace {transaction_reference} with the Payd transaction reference returned from the original API call or webhook (e.g., 9BD103350408eR).

Reference suffixes
The suffix of a Payd transaction reference indicates the transaction type:

Suffix	Transaction type	Example
eR	Receipt (payin/collection)	9BD103350408eR
eW	Withdrawal (payout)	9BD090722409eW
eS	Payd-to-Payd transfer	9BD12041887.eS
eT	Top-up	9BD110045632eT

Response fields
Field	Type	Description
id	string	Internal unique identifier (UUID)
code	string	The Payd transaction reference
currency	string	Transaction currency (e.g., "KES", "NGN")
amount	number	Transaction amount
balance	number	Account balance after the transaction
type	string	Transaction type: "receipt", "withdrawal", "transfer", or "topup"
transaction_category	string	Category: "fiat" for local currency transactions
conversion_rate	number	FX conversion rate applied (0 if same currency)
transaction_details	object	Detailed transaction info (see below)
created_at	string	ISO 8601 timestamp of when the transaction was created
transaction_details
Field	Type	Description
status	string	Transaction status: "success" or "failed"
payer	string	Phone number or identifier of the payer
receiver	string	Name of the receiving party
phone_number	string	Phone number involved in the transaction
merchant_id	string	Payd reference combined with the third-party transaction ID
reason	string	Human-readable outcome description
channel	string	Payment channel (e.g., "mobile", "bank", "card")
processed_at	object	Timestamp with seconds (Unix epoch) and nanos
email_address	string	Email address associated with the account
account_number	string	Internal account identifier
